# 火山方舟 Agent Plan (AFP) 抵扣规则

> 来源：https://www.volcengine.com/docs/82379/2516283
> 验证日期：2026-06-29

## 概述

火山方舟 Agent Plan 是一种按 AFP（Agent Function Point）计费的套餐模式。
不同模型在不同上下文长度下有不同的 AFP 抵扣系数。

## AFP 计算公式

```
AFP = (input_tokens × input_coef + output_tokens × output_coef) / 10000
```

## 上下文分段系数

上下文长度按 **输入 token 数** 分段，不同分段使用不同的输入系数：

| 上下文区间 | 乘数 | 说明 |
|-----------|------|------|
| 0 - 32k | ×0.67 | 短上下文，享受折扣 |
| 32k - 128k | ×1.0 | 标准上下文 |
| >128k | ×2.0 | 长上下文，加收 |

注意：乘数仅影响输入系数，输出系数不分段。

## 计算示例

**示例 1：doubao-seed-2-0-mini，输入 50000 tokens，输出 2000 tokens**
- 上下文区间：32k-128k（50000 > 32768）
- 输入系数：0.25，输出系数：0.25
- AFP = (50000 × 0.25 + 2000 × 0.25) / 10000 = (12500 + 500) / 10000 = 1.30

**示例 2：deepseek-v4-flash，输入 20000 tokens，输出 1000 tokens**
- 上下文区间：0-32k（20000 ≤ 32768）
- 输入系数：0.335，输出系数：0.5
- AFP = (20000 × 0.335 + 1000 × 0.5) / 10000 = (6700 + 500) / 10000 = 0.72
